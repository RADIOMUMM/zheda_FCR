## 1. 概述

本文档补充说明 FCR 计算软件 v1.2 中**超出基础需求文档**的新增功能和改进特性。基础算法（5 步体重质控、10 种采食错误过滤、ADG/ADFI/FCR 计算）请参见《FCR计算软件需求文档.md》。

| 异常详情列 | "是否存在填补数据" | ✅ "异常详情" |

## 3. 批量输入功能

### 3.1 使用场景

当同一批次的数据分散在多个 CSV 文件中时（如不同栏位的测定站分别导出），可使用批量输入功能一次性加载。

### 3.2 操作步骤

1. 点击 **"浏览文件夹"** 按钮，选择包含 CSV 文件的文件夹
2. 软件自动扫描文件夹下所有 `*.csv` 文件
3. 所有 CSV 按 `lifenumber`（个体批次号）合并，同一头猪的数据自动归并
4. 合并进度显示在 **合并进度条** 上

### 3.3 数据合并规则

```
文件夹/
├── location51.csv    → 合并
├── location55.csv    → 合并      → 按 lifenumber 分组 → 逐头处理
├── location56.csv    → 合并
└── location57.csv    → 合并
```

- 各 CSV 必须包含相同的 9 列结构
- 同一头猪出现在多个 CSV 中时，记录按 visit_time 合并
- 每头猪的来源文件记录在 `_source_file` 字段，用于树形分组展示

### 3.4 单文件 vs 批量模式

| 对比项 | 单文件模式 | 批量模式 |
|--------|-----------|---------|
| 输入方式 | 浏览单个文件 | 浏览文件夹 |
| 合并进度条 | 不显示 | 显示 |
| 猪只列表分组 | 一个文件节点 | 多个文件节点 |


## 4. 阶段 FCR 计算

### 4.1 概念

阶段料肉比是对猪只在**特定体重区间内**（如 50-90kg）的 FCR 进行计算，排除区间外的数据干扰，更精确地反映该生长阶段的饲料转化效率。

### 4.2 启用方式

1. 勾选 **"阶段料肉比计算"** 复选框
2. 设置区间下限（默认 50 kg）和上限（默认 90 kg）
3. 上下限输入框在未勾选时禁用，勾选后可用

### 4.3 计算流程

```
勾选阶段计算 → process_pig 主流程完成后
    │
    ▼
从 QC 后数据中筛选 is_abnormal=False 且 qc_weight_kg 非空的记录
    │
    ▼
按体重区间 [stage_lower, stage_upper] 过滤
    │
    ▼
调用完整的 process_pig（含 5 步质控、ADG/ADFI/FCR）
    │
    ▼
↓ 质控参数：
  init_lower = stage_lower   # 阶段区间下限
  init_upper = stage_upper   # 阶段区间上限
  end_lower  = 90 (default)  # 终测体重下限
  end_upper  = 110 (default) # 终测体重上限
    │
    ▼
结果写入 PigResult.stage_* 字段
```

### 4.4 与全期 FCR 的区别

| 对比项 | 全期 FCR | 阶段 FCR |
|--------|----------|----------|
| 数据范围 | 该猪全部记录 | 仅体重在区间内的记录 |
| 初测范围检查 | 使用用户配置的初测范围 | 使用阶段区间作为初测范围 |
| 输出文件 | `02.样本肉料比信息表.xlsx` | `02.样本阶段肉料比（X-Xkg）信息表.xlsx` |
| 结果字段 | `result.fcr/adg/...` | `result.stage_fcr/stage_adg/...` |

### 4.5 阶段图片

勾选阶段计算时，会在输出目录的 `03.结果图片/` 中额外生成阶段图片：
- `{lifenumber}.png` — 全期图片
- `{lifenumber}_阶段({lo}-{hi}kg).png` — 阶段图片（区间内点蓝色高亮，区间外点灰色）

右侧预览区根据阶段复选框状态自动切换显示：
- **勾选时** → 优先显示阶段图片
- **取消勾选时** → 回退到全期图片


## 5. 体重质控增强

### 5.1 ADG 参考值 — 线性回归替代首尾法

**原方法（V1.0）：**
```
ADG_ref = (end_weight − start_weight) / (end_date − start_date)
// 仅使用首尾两条记录，易受异常值污染
```

**改进方法（V1.2）：**
```
对所有有效记录做最小二乘线性回归
ADG_ref = 回归斜率（m_reg）
start_wt = 回归截距（c_reg）  # 不受首条异常值影响
```

**效果：** 当数据末尾存在异常回落记录（如 64.1kg 异常值）时，回归斜率仍保持正确，不会被拉低。

### 5.2 起止体重提取 — 回归残差离群检测

**原方法（V1.0）：**
```
start_weight = raw_data.iloc[0]   # 直接从原始数据取首条
end_weight = raw_data.iloc[-1]    # 直接从原始数据取末条
// 异常尖峰/异常回落污染起止体重
```

**改进方法（V1.2）：**
```
从 QC 后有效记录提取：
1. 重新拟合趋势线
2. 计算每条记录的残差 |actual − predicted|
3. 若首条残差 > 8kg 且远大于第二条 → 跳过
4. 若末条残差 > 8kg 且远大于倒数第二条 → 跳过
```

**效果：** 体重在正式入试前出现的异常尖峰（如 62.2kg、69.8kg）会被正确跳过，起止体重取自正常的增长趋势起点。

### 5.3 初/终测体重范围检查

**原方法（V1.0）：**
```
使用固定常量 BW_RANGE_LOWER=45, BW_RANGE_UPPER=110
→ 与用户配置的 25-45 / 90-110 脱节
```

**改进方法（V1.2）：**
```
使用用户配置的 init_lower/init_upper（初测体重范围）
使用用户配置的 end_lower/end_upper（终测体重范围）
→ 与界面参数一致
```

**异常详情输出格式：**
```
初测体重37.6kg（正常范围25-45kg）   ← 统一格式，包含实际值
终测体重122.9kg（正常范围90-110kg）  ← 和配置范围
```

| 列名 | 说明 | 示例 |
|:----|------|:-----|
| 个体批次号 | 猪只标识 | L24.252601 |

阶段汇总表结构与全期汇总表一致，仅数据替换为阶段区间内的计算结果。阶段区间标记在"异常详情"列中：
```
阶段(50-90kg); 初测体重52.7kg（正常范围50-90kg）; ...
```


## 7. 常见问题

### 7.1 闪退/崩溃

如果软件闪退，V1.2 已添加全局异常捕获，会弹出错误对话框显示具体信息。常见原因：

| 问题 | 解决方法 |
|:----|:---------|
| CSV 文件路径包含中文 | 确保在 Windows 下运行，路径编码为 UTF-8 |
| CSV 格式不正确 | 检查列名是否匹配需求文档 |
| 输出目录无权限 | 更换输出路径 |

### 7.2 阶段起止体重不符合预期

如果阶段起始体重明显高于区间下限（如设置 50-90kg 但起始体重视为 62.2kg），通常是因为数据中存在异常尖峰记录（传感器误读），V1.2 的回归残差检测会自动跳过此类离群值。

### 7.3 图片中 R² 不显示

V1.2 已修复中文字体不支持上标 ² 的问题，图片中显示为 `R^2`。
