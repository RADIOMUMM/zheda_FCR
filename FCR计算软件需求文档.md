## 1. 项目概述

### 1.1 项目背景

本软件旨在实现浙江大学博士论文《猪饲料转化效率和采食行为的 GWAS 及基因组选择研究》中提出的 **FCR（Feed Conversion Ratio，料重比）计算方法**，将论文中的数据处理和计算流程转化为可交互、可复现的桌面应用程序。

### 1.2 核心目标

- 完整实现论文中 **5 步体重质控法** + **16 种错误类型过滤** + **FCR 计算**
- 提供直观的 **体重-日龄拟合曲线图**（含/不含拟合线，可切换）
- 输出 **每只猪的清洗后明细数据** 和 **全部猪的 FCR 汇总表**

### 1.3 适用范围

- **目标用户**：猪场生产管理人员、育种数据分析人员
- **输入数据**：电子饲喂站（如 FIRE 系统、9ZC-170 等）导出的原始 CSV 数据
- **输出格式**：Excel 文件（明细 + 汇总）+ PNG 图片（单猪体重曲线）

## 3. 数据处理与 FCR 计算算法（完全基于浙大博士论文方法）

### 3.1 算法总体流程

```
原始数据
    │
    ▼
[Step 1] 个体层面预过滤（日龄、体重极值）
    │
    ▼
[Step 2] 采食数据过滤（Casey 2005 的 16 种错误类型）
    │
    ▼
[Step 3] 体重数据质控（5 步过滤法）← 本文核心创新
    │
    ▼
[Step 4] ADG 计算（线性回归）
    │
    ▼
[Step 5] ADFI 计算（平均日采食量）
    │
    ▼
[Step 6] FCR 计算（料重比）
    │
    ▼
[Step 7] 输出结果
```

### 3.2 Step 1：个体层面预过滤

| 规则 | 阈值 | 说明 |
|:----:|:----:|:-----|
| 测试日龄下限 | < 100 天 | 入试时体重 / ADG 推算的日龄 |
| 测试日龄上限 | > 200 天 | 超出正常测试周期 |
| 体重上限 | ≥ 180 kg | 超出正常生长范围 |
| 体重下限 | ≤ 25 kg | 低于正常入试体重 |

**实现逻辑**：
1. 对每头猪，从 visit_time 中提取最早和最晚访问时间
2. 从最早访问时的体重推算入试日龄（近似值，或从数据中直接计算）
3. 若入试日龄 < 100 或 > 200，整体标记为异常
4. 对超出体重阈值的 **单条记录** 直接剔除（不剔除整头猪）

### 3.3 Step 2：采食数据过滤 — 16 种错误类型

完全采用 **Casey, Stern, Dekkers (2005)** 定义的 16 种错误类型及阈值：

#### 3.3.1 计算 7 个衍生变量

对每条访问记录，计算：

| 变量 | 含义 | 计算公式 |
|:----:|------|:--------|
| FIV | 次采食量 (g) | entry_weight − exit_weight |
| OTV | 次采食时长 (s) | exit_time − entry_time |
| FRV | 采食速率 (g/min) | FIV / (OTV / 60)，OTV ≤ 0 时不计算 |
| LWD | 前向重量差 (g) | 后一次 entry_weight − 当前 exit_weight |
| FWD | 后向重量差 (g) | 当前 entry_weight − 前一次 exit_weight |
| LTD | 前向时间差 (s) | 后一次 entry_time − 当前 exit_time |
| FTD | 后向时间差 (s) | 当前 entry_time − 前一次 exit_time |

#### 3.3.2 16 种错误类型判定

| 序号 | 错误类型 | 变量 | 适用范围 | 阈值 |
|:----:|:--------:|:----:|:---------|:----:|
| 1 | FIV-lo | FIV | 所有访问 | FIV < -20 g |
| 2 | FIV-hi | FIV | 所有访问 | FIV > 2,000 g |
| 3 | FIV-0 | FIV | OTV = 0 s 的访问 | |FIV| > 20 g |
| 4 | OTV-lo | OTV | 所有访问 | OTV < 0 s |
| 5 | OTV-hi | OTV | 所有访问 | OTV > 3,600 s |
| 6 | FRV-hi-FIV-lo | FRV | 0 < FIV < 50 g | FRV > 500 g/min |
| 7 | FRV-hi-strict | FRV | FIV ≥ 50 g 且前后有 FIV < -20 g | FRV > 110 g/min |
| 8 | FRV-hi | FRV | FIV ≥ 50 g 且前后无 FIV < -20 g | FRV > 170 g/min |
| 9 | FRV-0 | FRV | FRV = 0 g/min | OTV > 500 s |
| 10 | FRV-lo | FRV | FRV ≠ 0 g/min | |FRV| ≤ 2 g/min |
| 11 | LWD-lo | LWD | 除最后一次访问外 | LWD < -20 g |
| 12 | LWD-hi | LWD | 除最后一次访问外 | LWD > 1,800 g |
| 13 | FWD-lo | FWD | 除第一次访问外 | FWD < -20 g |
| 14 | FWD-hi | FWD | 除第一次访问外 | FWD > 1,800 g |
| 15 | LTD-lo | LTD | 除最后一次访问外 | LTD < 0 s |
| 16 | FTD-lo | FTD | 除第一次访问外 | FTD < 0 s |

**实现逻辑**：
- 对每条记录逐一判断上述 16 种错误类型
- 若某条记录命中任一错误类型，标记 `feed_error = True`
- 错误标记用于后续的采食数据备注，指示该条记录在采食量计算中是否被排除

#### 3.3.3 采食量备注规则

| 条件 | 备注内容 |
|:----|:---------|
| 无错误标记 | None（空） |
| 命中至少一个错误 | "采食数据异常：FIV-lo/FIV-hi/错误类型名称..." |

### 3.4 Step 3：体重数据质控 — 5 步过滤法（本文关键创新）

针对 FIRE 体重数据中常见的 **极端值** 和 **体重分层** 问题：

#### 第 1 步：粗过滤
- 剔除体重 > 180 kg 或 < 25 kg 的原始记录

#### 第 2 步：计算 ADG 参考值
```
ADG_ref = (end_weight − start_weight) / (end_age − start_age)
```
其中：
- start_weight：开测体重（站外校正秤测量，取最早的稳定读数）
- end_weight：结测体重（站外校正秤测量，取最晚的稳定读数）
- start_age / end_age：对应的测定日龄

#### 第 3 步：计算预测体重
```
predicted_weight = start_weight + (test_age − start_age) × ADG_ref
```
对每一条访问记录，根据其访问时间和计算出的 ADG_ref，推算当天的预期体重。

#### 第 4 步：极端错误过滤
- 剔除超出 `predicted_weight ± ADG_ref × 21` 范围的数据点
- 21 天被论文作者作为合理波动的最大天数窗口

#### 第 5 步：标准差二次过滤
- **(a) 2 倍标准差过滤**：对当天所有体重记录，剔除超出 `当天均值 ± 2 × SD` 的记录
- **(b) 7 倍 ADG 过滤**：对当天所有体重记录，剔除超出 `当天均值 ± ADG_ref × 7` 的记录

#### 体重数据备注规则

| 条件 | 备注内容 |
|:----|:---------|
| 未命中任何过滤规则 | None（空） |
| 仅命中第 5b 步（7 倍 ADG 过滤） | "回归偏差较大" |
| 命中其他过滤规则 | 根据命中的步骤具体描述 |

### 3.5 Step 4：ADG 计算

- **方法**：体重（BW，单位为 kg）对日龄（age，从 visit_time 提取）做 **线性回归**
- 回归系数（斜率）即为 **ADG**
- 回归使用所有 **通过 5 步体重质控** 的站内体重记录
- 站外开测体重和结测体重作为回归的锚定点

**实现细节**：
- 体重转换为千克（原始 CSV 为克）
- 日龄从 visit_time 中提取
- 线性回归使用最小二乘法
- 回归结果需给出 R² 作为拟合优度指标

### 3.6 Step 5：ADFI 计算

```
ADFI = 总有效采食量 / 测试天数
```

其中：
- **总有效采食量** = 所有未标记采食错误的访问的 feed_intake 之和（需换算为千克）
- **测试天数** = (最晚访问日期 − 最早访问日期).days

### 3.7 Step 6：FCR 计算

```
FCR（料重比）= ADFI / ADG
```

- FCR = Feed:Gain（饲料消耗量 / 体重增量），即传统料重比
- 以 kg/kg 为单位
- 保留 **3 位小数**

### 3.8 额外指标：原始 FCR

**原始 FCR** = 使用最小二乘法的截距和斜率计算整体料重比，公式为：
```
原始 FCR = 总体总采食量 / (ADG × 测试天数)
```
（具体参照已有实现中的 `原始FCR` 列的计算逻辑）

### 4.4 体重-日龄拟合曲线图

**图表规格**：
- 文件格式：PNG，分辨率 ≥ 150 DPI
- 图表尺寸：宽 800px × 高 500px（或可缩放矢量格式）

**图表内容**：
- **X 轴**：日期（Date），格式如 `02-22`
- **Y 轴**：体重（kg），范围根据数据自动适配
- **散点**：所有站内体重记录（蓝色圆点），标记通过质控的体重数据
- **拟合线**：ADG 线性回归拟合线（红色实线），可选显示/隐藏
- **文本标注**（图上或图下方）：
  - 猪只批次号
  - 起始日期、起始体重
  - 终止日期、终止体重
  - FCR 值
  - ADG 值
  - R² 值

**交互要求**：
- 提供复选框（Checkbox）：**"显示拟合曲线"**，默认勾选
- 取消勾选时仅显示散点，不显示拟合线
- 预览区域可缩放（鼠标滚轮或触控板缩放）

## 6. 异常处理与边界条件

| 场景 | 处理方式 |
|:----|:---------|
| CSV 文件格式不正确 | 弹出错误提示，指明缺少的列名 |
| 猪只数据不足 20 条 | 标记为数据不足，不参与 FCR 计算 |
| 测试天数不足（< 30 天） | 在汇总表"是否存在填补数据"列中标注 |
| 体重范围异常（不在 45-110 kg 之间） | 在"是否存在填补数据"列中标注 |
| 无有效体重数据（全部被质控过滤） | 标记为异常样本，FCR 留空 |
| ADG 回归 R² < 0.8 | 标记为拟合质量偏低预警（非强制剔除） |
| 文件同名冲突 | 自动添加时间戳后缀避免覆盖 |
